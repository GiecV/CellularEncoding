tasks.cartpole module

tasks.cartpole.compute_fitness(individual)[source]

Compute the fitness of an individual in the CartPole environment.

The fitness is measured by the mean reward obtained over a number of trials, where each trial consists of running the individual’s neural network in the CartPole-v1 environment.

Parameters:

individual (Any (typically an individual in an evolutionary algorithm)) – The individual whose fitness is to be computed. It is expected to be an object that can be converted into a Phenotype for neural network evaluation.

Returns:

The mean reward obtained over a number of trials.

Return type:

float

tasks.cartpole.compute_fitness_growth_penalty(individual, penalty=0)[source]

Compute the fitness of an individual with a growth penalty.

This function evaluates the fitness of an individual by first calculating its original fitness using compute_fitness. Then, it applies a penalty based on the number of nodes in the individual’s genotype. The penalty helps to discourage overly complex solutions.

Parameters:

  • individual (Any (typically an individual in an evolutionary algorithm)) – The individual whose fitness is to be computed.

  • penalty (float, optional) – The penalty factor for the number of nodes. Default is 0, meaning no penalty is applied.

Returns:

The penalized fitness value.

Return type:

float

tasks.parity module

tasks.parity.compute_fitness(individual, n=2)[source]

Compute the fitness of an individual in the parity problem.

The fitness is evaluated by computing the normalized mutual information score between the outputs of a neural network and the expected parity targets for all possible binary input combinations. The network is evaluated with n binary inputs, and the fitness is a measure of how well the network predicts the parity of the inputs.

Parameters:

  • individual (Any (typically an individual in an evolutionary algorithm)) – The individual whose fitness is to be computed. It is expected to be an object that can be converted into a Phenotype for neural network evaluation.

  • n (int, optional) – The number of binary inputs to the neural network. Default is 2.

Returns:

The normalized mutual information score between the network’s outputs and the expected targets.

Return type:

float

tasks.parity.compute_fitness_range(individual, n=5)[source]

Compute the fitness of an individual in the parity problem for a range of inputs.

The fitness is evaluated by computing the normalized mutual information score between the outputs of a neural network and the expected parity targets for all possible binary input combinations. The network is evaluated with binary inputs ranging from 2 to n, and the fitness is a measure of how well the network predicts the parity of the inputs.

Parameters:

  • individual (Any (typically an individual in an evolutionary algorithm)) – The individual whose fitness is to be computed. It is expected to be an object that can be converted into a Phenotype for neural network evaluation.

  • n (int, optional) – The maximum number of binary inputs to the neural network. Default is 5.

Returns:

A list of normalized mutual information scores for each input size.

Return type:

list of float

tasks.xor_gate.compute_fitness(individual)[source]

Compute the fitness of an individual for the XOR problem.

The fitness is calculated by comparing the neural network’s outputs with the expected results for all possible input combinations. The fitness value is the proportion of correct outputs for the XOR problem.

Parameters:

individual (Any (typically an individual in an evolutionary algorithm)) – The individual whose fitness is to be computed. It is expected to be an object that can be converted into a Phenotype for neural network evaluation.

Returns:

The fitness value, which is the proportion of correct outputs.

Return type:

float

tasks.xor_gate.compute_fitness_information_formula(individual)[source]

Compute the fitness of an individual using an information-theoretic formula.

The fitness is based on the mutual information between the neural network’s outputs and the expected targets, with a regularization term for the complexity of the network. This combines the accuracy of the network’s predictions and its complexity to provide a more robust measure of fitness.

Parameters:

individual (Any (typically an individual in an evolutionary algorithm)) – The individual whose fitness is to be computed.

Returns:

The fitness value based on mutual information and neural network complexity.

Return type:

float

tasks.xor_gate.compute_fitness_target(individual, print_info=False)[source]

Compute the fitness of an individual with an option to print detailed information.

This function computes the fitness by comparing the individual’s tree structure to a target tree structure. The fitness is based on several factors including depth, number of nodes, common tags, and successors. The more similar the individual’s tree structure is to the target, the higher the fitness.

Parameters:

  • individual (Genome) – The individual whose fitness is to be computed. It is expected to be an object that can be converted into a Genome and compared with a target genome.

  • print_info (bool, optional) – Whether to print detailed information about the fitness calculation. Default is False.

Returns:

The fitness value based on the similarity of the tree structures.

Return type:

float
